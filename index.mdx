## Endpoint

**Production:** `https://rpc.orderpath.xyz`

OrderPath uses standard JSON-RPC 2.0, so you can use it as a drop-in replacement for any Ethereum RPC endpoint.

## Submit a Transaction

Send a signed raw transaction using `eth_sendRawTransaction`:

```bash
curl -X POST https://rpc.orderpath.xyz \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "eth_sendRawTransaction",
    "params": ["0x02f8..."],
    "id": 1
  }'
```

Response:

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": "0x392d359de9fcc4d4164615ed5a2e09a1efea27f4d18f44ebb73643050bcafe24"
}
```

## Transaction Validation

Validate a transaction before submitting:

```bash
curl -X POST https://rpc.orderpath.xyz/validate-tx \
  -H "Content-Type: application/json" \
  -d '{
    "rawTx": "0x02f8..."
  }'
```

## Health Check

## Supported Methods

OrderPath supports all standard Ethereum JSON-RPC methods. Unsupported methods are automatically forwarded to an upstream RPC provider.

**OrderPath-specific methods:**

- `eth_sendRawTransaction` - Submit transaction to MEV builder

## Error Handling

All errors follow JSON-RPC 2.0 error format:

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32000,
    "message": "Transaction validation failed: invalid signature"
  }
}
```

Common error codes:

- `-32700` - Parse error
- `-32600` - Invalid Request
- `-32601` - Method not found
- `-32602` - Invalid params
- `-32000` - Server error (validation, submission, etc.)
